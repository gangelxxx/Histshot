# Текущее состояние проекта Histshot (v0.1.0)

## Общая информация

- **Название:** Histshot.
- **Репозиторий:** `d:\programming\my\temp\iscreen` (GIT-дерево чистое, последний коммит `7dddc75`, метка `1`).
- **Версия:** `0.1.0` (задана в `src/Histshot/Histshot.csproj`, CI может переопределять через git-тег).
- **Лицензия:** MIT.
- **Назначение:** лёгкое кроссплатформенное десктоп-приложение для создания скриншотов: живёт в системном трее, выделяет область экрана, позволяет аннотировать (карандаш, линия, стрелка, текст) и копировать/сохранять результат, ведёт локальную историю.
- **Целевая ОС:** Windows (захват экрана реализован только для Windows); UI-часть на Avalonia потенциально кроссплатформенна, но macOS/Linux требуют своего `IScreenCaptureProvider`.

## Технологический стек

- **Язык:** C# 12, .NET 8 (`net8.0`).
- **UI-фреймворк:** Avalonia UI 12.0.4 + FluentTheme + Inter font.
- **Рендеринг/изображения:** SkiaSharp 3.119.4 (`Avalonia.Skia`, прямое использование `SKBitmap`/`SKCanvas`).
- **MVVM/команды:** CommunityToolkit.Mvvm 8.4.1 (`RelayCommand`).
- **DI:** Microsoft.Extensions.DependencyInjection 8.0.0.
- **Иконки:** Optris.Icons.Avalonia.MaterialDesign 12.0.7.
- **Автообновления:** Velopack 1.2.0 + `vpk` CLI; источник `https://github.com/gangelxxx/Histshot`.
- **Тесты:** xUnit 2.5.3 + Coverlet 6.0.0 + .NET Test SDK 17.8.0.
- **Система памяти проекта:** `cm.exe` (project memory & code map), данные в `memory/`, конфиг `config.json`.

## Пользовательский сценарий и поток данных

1. Приложение запускается и сидит в трее (`ShutdownMode.OnExplicitShutdown`).
2. Пользователь кликает по иконке трея или нажимает `Prnt Scrn`:
   - `App.StartCapture()` получает список мониторов через `IScreenCaptureService`.
   - Для каждого монитора создаётся `CaptureOverlayWindow`, которому передаётся захваченное изображение этого монитора (`CaptureScreenAsync`).
   - Оверлеи вызывают `StartCapture()`, где окно сначала невидимо, затем через `SetWindowPos` (двойной вызов) растягивается на физические границы монитора, после чего появляется.
3. Пользователь выделяет область мышью:
   - Рисуется рамка с 8 маркерами; область вне рамки затемнена.
   - После отпускания кнопки (если область > 5×5) включается режим редактирования.
4. Редактирование происходит **внутри оверлея**:
   - `EditorControl` загружает вырезанный кусок (`CaptureSelectionAsync`), масштабирует под `RenderScaling`.
   - `ToolbarControl` позволяет выбрать инструмент, цвет, толщину/шрифт.
   - Поддерживается рисование, стрелки, линии, текстовые блоки (с ручками перемещения и ресайза), выделение/перемещение/удаление объектов.
5. По кнопке «Copy» (`Ctrl+C`) или тулбара:
   - `EditorControl.RenderFinalImage()` рендерит итоговое изображение со всеми аннотациями.
   - `ClipboardService.SetImageAsync` копирует PNG в буфер обмена.
   - `HistoryService.SaveAsync` сохраняет PNG и миниатюру в историю.
   - Оверлеи закрываются (`CloseRequested`).
6. Альтернативно, «Quick save full screen» (`Shift + Prnt Scrn`) захватывает все мониторы сразу (`CaptureAllScreensAsync`), копирует в буфер и сохраняет в историю без UI.
7. История доступна через правый клик по трею → `HistoryWindow`; оттуда можно открыть, скопировать или удалить.

## Особенности реализации и текущие нюансы

- **Многомониторность и DPI:** реализован сложный workaround в `CaptureOverlayWindow`:
  - Двойной вызов `SetWindowPos` для обхода `WM_DPICHANGED`, который Avalonia обрабатывает ужатием окна.
  - Размеры контента (`Width`/`Height`, фоновое изображение) вычисляются от `this.RenderScaling`, а не от переданного `_scaling`, чтобы соответствовать реальному рендерингу окна.
  - Фоновый битмап строится один раз через `WriteableBitmap` (копирование пикселей из `SKBitmap`), а не через PNG-кодирование/декодирование.
- **Окно `EditorWindow`:** подготовлено и создано в `OnCaptured`-колбэке, но сам колбэк нигде в `CaptureOverlayWindow` не вызывается; весь пользовательский путь редактирования идёт внутри оверлея. Это фактически «мёртвый» код на случай будущего перехода.
- **Глобальные хоткеи:** Windows-only, парсер понимает модификаторы + `Prnt Scrn`/`F1`–`F12`; обычные буквы/цифры не поддерживаются.
- **Локализация:** кастомная система без `.resx`; язык применяется при старте и при сохранении настроек; окна пересоздаются, поэтому перевод подхватывается.
- **Автозапуск:** запись в `HKCU\Software\Microsoft\Windows\CurrentVersion\Run`.
- **Обновления:** Velopack, пункт трея «Update» активируется только после фоновой загрузки готового обновления.
- **Логирование:** `DebugLogger` пишет в `%LocalAppData%\Histshot\debug.log`; используется при диагностике DPI и захвата.

## Что требует внимания / следующие возможные шаги

- **Кроссплатформенный захват:** macOS и Linux требуют реализации `IScreenCaptureProvider`.
- **Использование `EditorWindow`:** либо удалить, либо задействовать для редактирования вместо встроенного оверлея.
- **Расширение парсера хоткеев:** поддержка буквенно-цифровых клавиш.
- **Производительность:** для 4K+ изображений стоит следить за утечками/частыми пересозданиями битмапов.
- **Переименование:** в `bin/obj/publish` ещё встречаются артефакты со старым именем `iScreen`.
