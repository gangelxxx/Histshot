# План реализации приложения для скриншотов

## Общая архитектура

Приложение запускается один раз, сворачивается в системный трей и ждёт команды на захват экрана. Основные компоненты:

- **TrayManager** — управление иконкой в трее, контекстным меню и историей.
- **CaptureManager** — затемнение экранов, выделение области, создание скриншота (должен запускаться мгновенно).
- **EditorWindow** — окно редактирования с панелью инструментов и холстом.
- **ToolSet** — реализация инструментов: карандаш, текст, стрелка, линия.
- **ToolSettings** — настройки инструментов: цвет, размер шрифта, толщина линий.
- **HistoryStorage** — сохранение и управление историей скриншотов.
- **ClipboardManager** — копирование результата в буфер обмена.

## Технологический стек

Учитывая требование кроссплатформенности (Windows, macOS, Linux), выбираем стек с нативной поддержкой всех трёх систем.

### Рекомендуемый вариант

- **Язык:** C# / .NET 8
- **UI-фреймворк:** Avalonia UI
  - Полноценная поддержка Windows, macOS и Linux.
  - Похожая на WPF модель (XAML, MVVM, Canvas, DrawingContext).
  - Поддержка SkiaSharp для отрисовки графики.
- **Графика:** Avalonia DrawingContext + SkiaSharp для сложной отрисовки.
- **Системный трей:** пакет `Avalonia.Labs.Notifications` или нативные биндинги (Windows/macOS/Linux tray).
- **Хранение истории:** SQLite через `Microsoft.Data.Sqlite` или JSON-файлы + папка с миниатюрами.

### Альтернативные варианты

| Стек | Плюсы | Минусы |
|------|-------|--------|
| **C# + Avalonia UI** | Один код на все ОС, XAML, .NET | Меньше готовых примеров, чем у WPF |
| **Python + PyQt / PySide** | Быстрая разработка, Qt кроссплатформенный | Медленнее, сложнее распространять |
| **Electron + TypeScript** | Привычный веб-стек, много библиотек | Высокое потребление ресурсов |
| **Flutter Desktop** | Единый UI на все платформы | Захват экрана и трей — через плагины |
| **Java + JavaFX** | Кроссплатформенность, зрелость | Большой размер, устаревший UI |

Лучший выбор для вашего проекта — **C# + Avalonia UI**: современный, быстрый, кроссплатформенный и хорошо подходит для десктопных приложений с кастомной графикой.

## Этапы реализации

### Этап 1. Подготовка проекта
- Создать решение и структуру проектов (Desktop- приложение, библиотеки ядра и тесты).
- Настроить MVVM-инфраструктуру (команды, уведомления об изменениях).
- Добавить базовые пакеты: DI-контейнер, SQLite (при необходимости).
- Реализовать запуск приложения в один экземпляр.

### Этап 2. Системный трей
- Создать `TrayManager` с иконкой приложения.
- Реализовать одиночный клик для запуска захвата.
- Реализовать правый клик с контекстным меню:
  - История скриншотов.
  - Выход.
- Обеспечить минимальное потребление ресурсов в режиме ожидания.

### Этап 3. Захват экрана
- Определить все подключённые мониторы и их разрешения/позиции.
- Создать полупрозрачные overlay-окна поверх всех экранов.
- Реализовать выделение области мышью с контрастной рамкой.
- Сделать двойной клик для быстрого захвата всего экрана (опционально).
- После отпускания кнопки мыши — получить bitmap выделенной области.
- Закрыть overlay и передать изображение в редактор.
- **Производительность:** от клика по трею до появления overlay не должно пройти более 100–200 мс. Для этого приложение работает в фоне, а overlay-окна создаются максимально легковесно.

### Этап 4. Редактор скриншотов
- Создать окно редактора с панелью инструментов.
- Реализовать холст для отображения скриншота.
- Реализовать инструменты:
  - **Карандаш** — свободное рисование линий.
  - **Текст** — размещение текстовой подписи с указанием позиции.
  - **Стрелка** — рисование стрелки от начальной до конечной точки.
  - **Линия** — рисование прямой линии.
- Поддержать масштабирование/прокрутку изображения при необходимости.

### Этап 5. Настройка инструментов
- Добавить панель/выпадающие настройки для текущего инструмента:
  - Выбор цвета (color picker).
  - Толщина линий (slider или preset-значения).
  - Размер шрифта для текстового инструмента.
- Хранить текущие настройки в `ToolSettings` и применять при рисовании.
- Предусмотреть значения по умолчанию.

### Этап 6. Сохранение и буфер обмена
- Добавить кнопки «Сохранить» и «Копировать в буфер обмена».
- При копировании — экспортировать изображение в Clipboard.
- При сохранении — сохранить файл в выбранную папку (png/jpg).
- После действия закрыть редактор и вернуть приложение в трей.

### Этап 7. История скриншотов
- Реализовать `HistoryStorage`:
  - Сохранение метаданных (время, путь к файлу/миниатюре).
  - Генерация миниатюр.
- Добавить меню истории в трее с миниатюрами и временем создания.
- Реализовать действия для элемента истории:
  - Открыть в редакторе.
  - Копировать повторно.
  - Удалить из истории.

### Этап 8. Тестирование и финальная доводка
- Проверить работу на нескольких мониторах с разным DPI.
- Проверить обработку отмены захвата (Escape).
- Протестировать все инструменты и настройки.
- Проверить историю, копирование и сохранение.
- Добавить обработку ошибок и логирование.

## Возможные риски

- Различия в DPI между мониторами могут искажать координаты захвата.
- Некоторые оверлей-окна могут не корректно перекрывать полноэкранные приложения.
- Производительность рисования на больших изображениях.
- **Кроссплатформенные отличия:**
  - API захвата экрана различается на Windows, macOS и Linux.
  - Системный трей и горячие клавиши требуют платформенных реализаций.
  - Доступ к буфер обмена и форматам изображений отличается.
  - На macOS может потребоваться разрешение на запись экрана (Screen Recording permission).
- Avalonia UI для некоторых нативных функций (трей, глобальные хоткеи) может требовать дополнительных библиотек.

## Критерии готовности

- [ ] Приложение запускается и работает из трея.
- [ ] Захват области работает на всех мониторах.
- [ ] Редактор позволяет рисовать всеми четырьмя инструментами.
- [ ] Доступны настройки цвета, толщины линий и размера шрифта.
- [ ] Скриншот копируется в буфер обмена.
- [ ] История сохраняется и отображается в трее.
- [ ] Ресурсы в режиме ожидания минимальны.
